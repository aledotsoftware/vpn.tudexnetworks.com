Se agregaron logs adicionales de auditoría en `config/entrypoint.sh` para la generación de la llave de API y la llave Auth de satélites. También se añadieron logs para recuperación de identidad (`IDENTITY_RECOVERY`) y validación de API Key (`API_KEY_RECOVERY`).

  - **Hardening HAProxy**: Se añadieron protecciones removiendo la cabecera `X-Powered-By` e incorporando una página de error HTTP 401 explícita.
- **DevOps Mesh Orchestrator Agent**: He optimizado la infraestructura y el despliegue del gateway.
  - **Docker Optimization**: Se migró de `alpine:latest` a `alpine:3.19` en el Dockerfile para garantizar determinismo y cacheo eficiente. Las instrucciones `RUN` se consolidaron en una sola capa reduciendo la fragmentación de la imagen. Además, se añadió un archivo `.dockerignore` exhaustivo para reducir el contexto de build y optimizar el tamaño y tiempo de generación de la imagen. **Novedad**: Se implementó un patrón *multi-stage build* (`AS builder`) en el Dockerfile para descargar dependencias pesadas (como el binario de Headscale) en una capa separada y mantener la imagen final lo más liviana y limpia posible.
  - **Entrypoint Robustness**: Se añadió lógica de reintentos (hasta 30 reintentos de 2 segundos) a la espera de `mariadb-admin ping` en `config/entrypoint.sh`, agregando un comando de `timeout` para prevenir cuelgues de red infinitos. También se mejoró la captura de señales en `trap` para emitir un `kill` explícito a los procesos hijos en background (como `headscale` y `tailscale`) asegurando que el contenedor salga de manera verdaderamente limpia. **Novedad**: Se reemplazó el obsoleto comando estático `sleep 10` para la inicialización de Headscale por un *active polling* inteligente. Ahora el script hace consultas continuas con `curl` al puerto de métricas (`9090`) de Headscale para asegurar que esté listo y operativo antes de proceder con el aprovisionamiento de claves, lo que resulta en un arranque del gateway mucho más robusto.
  - **CI/CD Readiness**: Se optimizó la rutina de prueba de infraestructura (`scripts/test-infra.sh` y `.github/workflows/ci.yml`) para ejecutar HAProxy de forma local en lugar de depender de imágenes de Docker Hub, previniendo fallos en CI debidos a restricciones de límite de peticiones (rate limits) y reduciendo los tiempos de ejecución del pipeline.
  - **Database Sync Optimization**: Se auditó el esquema de base de datos (`database/schema.sql`). Para prevenir cualquier problema de corrupción de secretos guardados (como en bases de datos que ignoran la sensibilidad a mayúsculas), la columna `key_content` en `headscale_secrets` ahora usa `CHARACTER SET utf8mb4 COLLATE utf8mb4_bin`.
